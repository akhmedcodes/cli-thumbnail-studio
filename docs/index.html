Programmatic API

Use directly in Python

Every feature is accessible as a clean Python API — no CLI required for scripting.

Python 
from core.slide_config import SlideConfig
from core.slide_renderer import render_slide_and_save

cfg = SlideConfig(
    layout_type="code_focus",
    theme="coding_tutorial",
    title="FastAPI Route Handler",
    subtitle="async def with type hints",
    code_snippet="""from fastapi import FastAPI
app = FastAPI()

@app.get('/hello')
async def hello():
    return {'message': 'Hello World'}""",
    code_language="python",
    width=1920, height=1080,
)

path = render_slide_and_save(cfg)
print(f"Saved: {path}")   # output/slide_20260518_175610.png

Installation

Up and running in 60 seconds

Requires Python 3.8+. Works on Linux, macOS and Windows.

1

Clone the repository

$ git clone https://github.com/akhmedcodes/cli-thumbnail-studio.git
$ cd cli-thumbnail-studio
2

Install dependencies

$ pip install Pillow rich colorama
# Optional — makes gradients ~10× faster
$ pip install numpy
3

Run

$ python3 run.py

Or use a shortcut flag:

$ python3 run.py --tool 2          # slide generator
$ python3 run.py --demo             # demo gallery
$ python3 run.py --batch-themes     # all themes at once
$ python3 run.py --preview          # localhost:7771

FAQ

Frequently asked questions

Is NumPy required?
No. NumPy is completely optional. When installed it makes gradient rendering approximately 10× faster. When absent, the tool automatically falls back to a pure-PIL implementation that works on any Python installation.
What fonts does it use?
The font manager searches your system for common fonts (DejaVu, Ubuntu, Roboto, Arial, Impact, JetBrains Mono, etc.) across Linux, macOS and Windows paths. You can also drop any .ttf file into assets/fonts/ and it will be discovered automatically. If no font is found, Pillow's built-in scalable font is used as a fallback.
Can I use it non-interactively / in scripts?
Yes. Every generator has a clean Python API. Import SlideConfig and render_slide_and_save() (or ThumbnailConfig and render_and_save()) directly from your code. See the code examples above.
What image formats does it export?
PNG with full quality (lossless). Files are saved to the output/ directory with an auto-generated timestamp in the filename, e.g. thumbnail_20260518_175610.png.
How does the browser preview work?
Running python3 run.py --preview starts a lightweight HTTP server on localhost:7771 using Python's built-in http.server — no extra packages needed. The page polls /meta every 2 seconds and reloads the image when a new file is detected. The server runs as a daemon thread and automatically shuts down when the process exits.
Does batch rendering use multiple CPU cores?
Yes. The batch renderer uses concurrent.futures.ThreadPoolExecutor with 4 worker threads by default. Because Pillow releases the GIL during image operations, multiple renders run in parallel with a real speedup on multi-core machines.
Can I add my own themes?
Yes. Open core/slide_themes.py (or core/themes.py for thumbnails) and add a new entry to the SLIDE_THEMES / THEMES dict following the same key structure as the existing themes. Your theme will automatically appear in the CLI menu.
